업로드 변경 유형 (Diff 로직)

유형색상판정 기준
NEW초록Excel에는 있지만 DB에 없는 SO (so_number + sku_code + line_item 조합 최초)
MODIFIED노랑DB에 이미 있고 아래 비교 필드 중 하나 이상 변경됨
UNCHANGED없음DB와 완전히 동일. 미리보기 탭에 표시 안 됨
CLOSED빨강DB에 있지만 업로드 Excel에 없는 SO → 자동 CLOSED
⚠ CLOSED 자동 처리 — 반드시 숙지
업로드한 Excel 파일에 포함되지 않은 기존 OPEN SO는 자동으로 CLOSED 처리됩니다.
예) 신규 SO 10건만 담은 Excel을 올리면 → 기존 OPEN SO 전체가 CLOSED.
항상 전체 OPEN SO가 담긴 파일을 올려야 합니다. 실수했다면 즉시 ↩ Rollback 사용.

다음 중 하나라도 다르면 MODIFIED로 분류되고 미리보기에 표시됩니다:

  • status — OPEN / HOLD / CLOSED
  • start_no_earlier — 최조기 시작 가능일

SO 분할 (Split)

하나의 SO Line Item 수량을 납기·우선순위가 다른 복수 Line Item으로 나눌 때 사용합니다.

분할할 SO 행 우클릭 → ✂ Split
SplitSODialog: 원본 행(파란 배경, Line Item 잠금) + 신규 행 자동 추가
신규 행에 수량·납기·우선순위 입력. 상단 Remaining 카운터가 0이 되어야 Split 버튼 활성화
✂ Split 확정 → 신규 Line Item 자동 채번 (예: L01-2, L01-3) 후 저장
분할 후 Execute Plan을 다시 실행하면 각 Line Item이 개별 우선순위로 독립 배정됩니다.

분할 자식(Split Child) 보호 — 업로드 시 자동 유지

분할로 생성된 Line Item(예: L01-2)은 내부 계획 결정이므로, ERP·고객사에서 받은 원본 Excel에는 포함되지 않는 것이 일반적입니다. 이 때문에 매번 업로드할 때마다 분할이 날아가는 문제를 방지하기 위해 분할 자식은 업로드 시 자동 CLOSED 대상에서 제외됩니다.

상황동작
Excel에 L01만 있고, DB에 L01(원본) + L01-2(분할)가 있는 경우 L01-2 보호 — CLOSED 처리 안 됨. 분할 상태 그대로 유지
Excel에서 L01의 qty·납기 등이 변경된 경우 L01은 MODIFIED ⚠ (N splits) 로 표시. 분할 수량을 수동으로 검토해야 함을 경고
Excel에서 L01의 status가 CLOSED로 명시된 경우 L01 CLOSED + L01-2 연쇄 CLOSED (의도적 종료이므로 자식도 닫힘)
Excel에 L01 자체가 없는 경우 (원본도 누락) L01 자동 CLOSED + L01-2 연쇄 CLOSED
UI에서 L01 우클릭 → 🚫 Close L01 CLOSED + L01-2 연쇄 CLOSED
업로드 미리보기에서 확인하는 법
부모 Line Item이 변경됐고 분할 자식이 있으면 Change 컬럼에 MODIFIED ⚠ (2 splits) 형태로 표시됩니다. 셀에 마우스를 올리면 "분할 자식은 자동 처리되지 않으니 수동 검토 필요"라는 툴팁이 뜹니다.
분할 후 부모 수량이 바뀐 경우
예) L01 qty=100 → L01 qty=60 + L01-2 qty=40으로 분할한 뒤, 고객이 qty를 80으로 줄여 Excel을 재발행하면 — L01은 MODIFIED(100→80)로 표시되고 L01-2는 보호되어 그대로입니다. 이 때 L01-2의 수량도 수동으로 조정해야 총합이 맞아떨어집니다. Upload Preview의 경고 표시를 보고 즉시 대응하세요.

Release Report

PLAN

SO·SKU·Line Item별 예상 출하 일정과 납기 준수 현황을 보여주는 리포트입니다.

Release Date 계산 기준

Release Date = 최종 공정(is_final_seq=1) 완료일 + SKU.post_lead_days

예: 최종 공정이 6/25 완료이고 post_lead_days=3이면 Release Date = 6/28.

상태 분류

상태기준
ON TIMERelease Date ≤ Due Date (여유 3일 초과)
AT RISKDue까지 3일 이내 여유
LATERelease Date > Due Date
NOT PLANNED계획 없음

필터

  • 검색창: SO번호 / SKU / Line으로 검색
  • 상태 필터: 드롭다운으로 특정 상태만 표시
  • 납기 N일 이내 필터
  • 📥 Export: 상태별 배경색 포함 Excel 내보내기

Dispatch List

PLAN

생산 현장에서 "지금 무엇을 먼저 만들어야 하는가"를 공정 단위로 보여주는 우선순위 큐입니다. Critical Ratio(CR) 기반으로 자동 정렬됩니다.

Critical Ratio (CR) 계산

CR = 납기까지 남은 오픈 Shift 수 ÷ 잔여 소요 Shift 수 (1 Room, hc_max 기준)

  • CR < 1.0 → 이미 납기 내 완료 불가능. 즉시 착수 필요
  • CR 1.0~1.5 → 위험 구간. 여유 없음
  • CR > 1.5 → 정상
True Slack = 오픈 Shift 수 − 소요 Shift 수. 음수면 이미 늦은 것입니다.

상태 색상

상태기준의미
CRITICALCR < 1.0납기 내 완료 불가 — 즉시 조치
AT RISKCR 1.0~1.5여유 없음 — 우선 감시
ON TIMECR > 1.5정상
COMPLETE실적 완료해당 공정 완료됨
NO ROUTING라우팅 미정공정 정보 없음 — 마스터 확인 필요

컬럼 설명

컬럼설명
Rank우선순위 순번 (CR 오름차순)
SO / Customer / SKU / Line오더 기본 정보

Rem. Qty잔여 생산 필요 수량

자주 묻는 질문

Execute Plan을 눌렀는데 계획이 생기지 않아요

  • CRP 경로가 설정되어 있는지 확인 (Masters → App Config)
  • 해당 날짜 캘린더가 Open인지 확인
  • Process Routing이 설정되어 있는지 확인 (no routing 상태 시 배정 불가)
  • SO 상태가 OPEN인지 확인 (HOLD/CLOSED 제외)

드래그로 이동했는데 블록이 사라졌어요

↩ Undo 버튼 또는 Ctrl+Z로 즉시 복구 가능합니다.

특정 공정으로 드래그할 때 빨간 고스트가 표시돼요

목적지 생산실이 해당 공정을 지원하지 않습니다. Room/Process 마스터에서 allowed_room_types를 확인하세요.

재고를 배정했는데 생산 계획 수량이 줄지 않아요

Execute Plan을 다시 실행하면 production_needed = SO qty − allocated − actual로 계산되어 계획 수량이 자동 조정됩니다.

FINAL 뱃지가 없는 공정 블록이 있어요

Process Routing에서 해당 SKU의 마지막 공정에 is_final_seq=1이 설정되어 있는지 확인하세요.

Dispatch List의 Shifts Needed가 너무 크게 나와요

CRP에서 해당 공정의 HC가 hc_max로 설정되어 있는지 확인하세요. Dispatch List는 1 Room·hc_max 기준으로 계산합니다. 여러 생산실을 병렬 사용하면 실제 소요 Shift는 더 짧아집니다.

SO를 분할했는데, 이후 고객이 원본 수량을 바꿔서 Excel을 다시 보냈어요. 어떻게 해야 하나요?

분할 자식(예: L01-2)은 업로드 시 자동 보호되므로 CLOSED되지 않습니다. 하지만 부모(L01)의 수량이 바뀌면 분할 자식의 수량은 자동으로 조정되지 않습니다. 아래 절차로 수동 정리하세요.

Upload Preview에서 확인 — 부모 L01이 MODIFIED ⚠ (N splits)로 표시되면 분할 자식이 있다는 뜻입니다. 셀 툴팁에 "수동 검토 필요" 안내가 표시됩니다.
총합 계산 — 변경된 부모 qty와 기존 분할 자식들의 qty 합계를 비교합니다.
예) 원본 100 → 고객이 80으로 변경. 기존 분할: L01=60, L01-2=40 (합계 100)
→ 20 감소분을 어느 Line Item에서 줄일지 결정 필요.
업로드 확정 — Confirm Upload로 부모 L01의 qty를 먼저 업데이트합니다.
분할 자식 수동 조정 — SO 탭에서 L01-2를 더블클릭 → SOEditDialog → qty 수정 후 저장.
또는 우클릭 → ✂ Split로 재분할할 수도 있습니다.
Execute Plan 재실행 — 수량이 정리된 뒤 Gantt → ⚡ Execute Plan으로 계획을 새로 반영합니다.
부모 qty가 증가한 경우도 동일합니다. 늘어난 수량을 기존 분할 행에 분배할지, 새 Line Item으로 추가 분할할지 플래너가 판단해야 합니다.
부모와 자식을 모두 없애고 처음부터 다시 분할하려면: 부모 L01을 CLOSED로 명시한 Excel을 올리면 L01과 모든 분할 자식이 연쇄 CLOSED됩니다. 이후 새 SO로 등록하거나 Rollback 후 재분할하세요.
